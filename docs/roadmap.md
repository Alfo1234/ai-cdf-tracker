AI-Powered CDF Tracker — Roadmap
Milestone 1 — Project Setup (Week 1)
Initialize GitHub repository
Create documentation (architecture, SRS, FRD, NFR)
Set up development environment (VS Code, Node.js, Python)
Install required tools (React, FastAPI, PostgreSQL)

Milestone 2 — Frontend UI Framework (Week 2–3)
2.1 Public Dashboard
Layout setup (Navbar, Footer, Routing)
Constituency list page
Constituency overview page
Interactive map integration
2.2 Admin Panel
Login page
Upload documents page
Project management view

Milestone 3 — Backend Core API (Week 4–5)
FastAPI setup
PostgreSQL database schema design
Endpoints for constituencies, projects, flags
Admin authentication
File upload endpoints

Milestone 4 — AI Data Extraction (Week 6–7)
PDF to text
OCR for scanned PDFs
NLP parsing (extract project name, cost, category)
Clean and validate extracted data

Milestone 5 — AI Summaries (Week 8)
Constituency-level summary generator
Project-level simplified explanations
Swahili version of summaries

Milestone 6 — Anomaly Detection Engine (Week 9–10)
Cost outlier detection
Missing fields detection
Cross-year comparison
Public Accountability Score formula

Milestone 7 — Integration (Week 11–12)
Connect frontend ↔ backend
Display summaries, flags, projects
Map visualization
Admin controls functional

Milestone 8 — Testing & QA (Week 13)
Unit tests (backend & AI models)
UI tests
Load testing

Milestone 9 — Deployment (Week 14)
Deploy backend to Render/railway
Deploy frontend to Vercel/Netlify
Set up a PostgreSQL cloud database

Milestone 10 — Final Polishing (Week 15)
UX improvements
Branding + UI cleanup
Documentation cleanup
Presentation-ready dashboard

Final Output
A fully working, visually professional AI-powered system that:
Extracts data from CDF documents
Summarizes them using AI
Detects anomalies
Displays results on a clean public dashboard
Provides admin control tools
Is deployable and presentable in hackathons or as a major project
